# 国旗クイズを Vercel に反映する手順書

対象アプリ：世界の国旗クイズ（kokki-quiz）  
使う道具：Cursor（または VS Code）と GitHub、Vercel  
目的：パソコンで直した内容を、インターネット上のクイズ画面に反映する

この紙を見ながら、上から順にやってください。

---

## 0. まず知っておくこと（とても大事）

パソコンの中のクイズと、Vercel のクイズは **別物** です。

| 場所 | 意味 |
| --- | --- |
| パソコン（Cursor） | いま直している場所 |
| GitHub | インターネット上の「保管箱」 |
| Vercel | みんなが見るホームページ |

流れはいつも同じです。

```
① 直す → ② Commit（記録） → ③ Push（送る） → ④ Vercel が自動更新
```

Push しないと、Vercel には反映されません。

---

## 1. 準備チェック

始める前に、次を確認します。

1. パソコンの電源が入っている
2. インターネットにつながっている
3. Cursor で **kokki-quiz** フォルダが開いている
4. GitHub にログインできる状態である

---

## 2. 変更を確認する

1. Cursor の左側で、**分岐のようなアイコン**（Source Control）をクリックする  
2. 「Changes」の下に、直したファイルが一覧で出る  
3. ここにファイルが出ていれば、今回送れる変更がある

もし何も出ていなければ、まだ保存されていないか、すでに送り済みです。

---

## 3. Commit（記録する）

### ステップ A：送るファイルを選ぶ

1. 「Changes」の横にある **＋** をクリックする  
2. 全部のファイルに ＋ が付いて、「Staged Changes」に移ればOK

### ステップ B：メッセージを書く

上のメッセージ欄に、何をしたか短く書きます。例：

```text
クリア状況のリセットと国旗えらびの画面を直した
```

### ステップ C：Commit を押す

1. **Commit** ボタンを押す  
2. エラーが出なければ成功  
3. これで「パソコンの中で記録完了」です（まだインターネットには届いていません）

---

## 4. Push（GitHub に送る）

1. **Sync Changes** または **Push**（雲や ↑ のアイコン）を押す  
2. ログインを求められたら、GitHub アカウントでログインする  
3. 少し待つ  
4. 左下や通知で「成功」と出ればOK

これで GitHub に届きました。

---

## 5. Vercel が自動で更新するのを待つ

1. そのまま **1〜2分** 待つ  
2. いつも見ているクイズのURLを開く  
3. 画面を再読み込みする（更新ボタン、または一度閉じて開き直す）  
4. 直した内容が見えれば成功

Vercel は、GitHub に新しいものが届くと、だいたい自動で作り直してくれます。

---

## 6. うまくいったかの見分け方

| 確認すること | OKの様子 |
| --- | --- |
| Commit | Source Control に変更が残っていない |
| Push | 「Push できない」「失敗」と出ていない |
| Vercel | 直した画面やボタンが見える |

今回の例（リセット機能）なら：

- マップ画面の上に **「クリア状況をリセット」** ボタンがある

---

## 7. うまくいかないときの対処

### Commit ボタンが押せない

- メッセージ欄が空になっていないか確認する
- ＋ でファイルを Staged にしたか確認する

### Push できない / ログインを求められる

- インターネットを確認する
- GitHub にログインし直す
- もう一度 Push を押す

### Vercel が古いまま

1. あと1〜2分待つ  
2. ページを再読み込みする  
3. スマホなら、一度タブを閉じて開き直す  
4. それでもだめなら、GitHub のページで最新の更新があるか見る

GitHub の場所（このプロジェクト）：

https://github.com/yonemasa89314-crypto/kokki-quiz

`main` ブランチに、いま書いたメッセージの Commit があれば、Push までは成功しています。

### 何を押していいか分からなくなった

この3つだけ覚えます。

1. **＋** で追加  
2. **Commit** で記録  
3. **Push / Sync** で送る  

---

## 8. よくある質問

### Q. Commit と Push の違いは？

- **Commit** … 自分のパソコンの中で「この変更を残す」  
- **Push** … その記録を GitHub（インターネット）へ送る  

Vercel が見るのは GitHub なので、**Push まで必要**です。

### Q. 友だちに送ったリンクでも、自分のクリア状況が見える？

基本的に見えません。クリア状況は、**その端末のブラウザの中**に保存されます。  
ただし、同じスマホ・同じパソコンで見せるときは、リセットボタンで消せます。

### Q. 途中で失敗したらどうする？

やり直して大丈夫です。  
もう一度 Source Control を開き、＋ → メッセージ → Commit → Push の順で進めてください。

---

## 9. いちばん短い覚え方

```
＋で追加 → メッセージを書く → Commit → Push → 1〜2分待つ → Vercelを開く
```

この紙を印刷して、机のそばに置いておくと安心です。
